关于我们

我们是一个大型开源组织 ApacheCN，旗下 QQ 群 9000 余人，订阅用户至少一万人。Github Star 数量超过 20k 个，在所有 Github 组织中排名前 150。网站日 uip 超过 4k，Alexa 排名的峰值为 20k。我们的核心成员拥有CSDN 博客专家和简书程序员优秀作者认证。


我们的人员构成，专科（主要以工作过为主）、本科（主要以在校生为主）、硕士（几乎很少），但是这群人有一个共性：热爱分享知识、并且想法很纯粹。
你可能会好奇为什么这群人能组建在一起，并且作出这样的成果？而其他组织没看到过？


专科（工作几年）：因为社会企业结构分化过于明显，学历偏低的人工作几年还能坚持学习和分享的人，是因为真的经历过非常艰难的考验：

1. 学历低机会少 (很多人当初选计算机，就是看到计算机看技术，而不是学历。但现实是在技术上的基础上加上学历的门槛，的确已经完全放弃了那些真正努力的人，这也就是为什么你看到的国内公司大佬学历都超级牛逼，因为不牛逼的基本上别人不会给机会你，所以活下来的已经基本上都是濒临灭绝。)

2. 面试过于苛刻（理论上是：学历越高，要求越严格。但现实社会和大家想的正好想法，我听得最多的是：他（学历高的人）技术差了一点，但是学历不错，可以培养；他（学历低的人）技术是还挺不错的，可以做事，但是学历太差，我们再看看有没有合适的，如果实在是没有，再考虑考虑，顺便再压压工资）

3. 资源过于匮乏（人脉资源基本为0，因为他们这个水平的也基本上都是做简单的技能，因为高级技能要看学历；人脉资源为0意味着，信息资源为0；所以能靠得住的是自学，所以为什么他们特别喜欢GitHub、Kaggle，而不是所谓的国家证书，因为他们知道，再技术面前，证书显得多么的苍白无力）

所以经历这么多，还乐意坚持学习和分享，我觉得真的不容易，这部分人适合做开源并且能长期坚持，因为他知道学习不容易、不希望其他的人也因为学历而失去了学习的机会（就好像经历过大风大浪的人，更加明白幸福和使命是什么一样）。
本科（在校学生）：这部分人很单纯、思想纯粹，没有被社会所摧残，保持着内心那美好的一部分，所以是乐于分享一些觉得好的东西，这个我觉得大家不会觉得奇怪，因为就是人最初的样子。

1. 兴趣爱好为主：没有受到过度的社会鄙视链，所以学习东西还是更具兴趣爱好去做，所以常常能做出一些东西，但是坚持不久。这是开源所需要的一部分就是资源信息的迭代，而这一部分他们却能是最大的贡献者之一。

2. 装逼分享为乐：人生最大的乐趣可能就是分享并且得到大家的认可，内心的那种充实感可能已经远远超出了学校的奖励。这也是开源对这部分群的吸引之处。

混入其他的人群：可能是感动，感动你们分享的资料、感谢你们的无私帮助，找到了当初那个默默无知被冷落 or 年少轻狂爱装逼 的自己，想起来当初的那个梦，所以也慢慢各个阶层的人参与进来。
而剩下的其他人：硕士、博士以就业为主，所以基本上都是学完就走（工资够他滚滚红尘，没必要做这些事情），对于企业工作几年的人来说，学历超过本科，基本上也就是代表这公司利益，所以开源存在自私的因素，很难做起来，也就是后天基因不够。还有一个原因薪资，基本上也可以混得还可以，没必要开源弄得那么苦逼，何必呢？还不如去滚滚红尘？？（当然也有个别特例，以上说笑，只是记录我看到的一部分群体，大家看看就行）
所以，基本上做开源：

1. 谈学历、谈资源、谈背景、基本上都做不起来。（开源的心不够纯粹）

2. 用人的标准只有一个：就是贡献度和纯粹的心。（开源最重要就是踏实做事和贡献）

3. 管理这一块，我们没什么大佬，期待有兴趣的大佬和我们一起来管理。

优化补充：（当然上面的话太粗糙的，行业大佬也不爱听，我们说点美化过的话）
在组织外部，我们与国内高质量公众号和知乎联盟合作互推，也与 AI 垂直领域 AI-Union 有内容上的整合和合作。在组织内部，我们积极吸收有意愿参与的贡献者，善于挖掘内部优秀的作者，例如：组织内部的学习直播分享，帮助组织内部人员做学习内容推广，对积极向上和乐于分享的人，赋予管理员/优秀群员职位（每月做红包奖励）。同时，我们组织了公益性的翻译活动、求职分享 和比赛活动，面向职场提供更有价值的干货，面向兴趣提供优质的内容平台，面向学习提供更具深度的资料，而不是简单的内容搬运。
与商业组织不同，我们并不会一味追逐热点更不会唯利是图。作为公益组织，我们将维护项目放在首要位置，旨在为国内AI社区提供真正有内容、有价值、有意义的高可用的项目，并有足够时间把项目打磨到极致。我们希望做出广大 AI 爱好者真正需要的东西，打造真正有价值的长尾作品。在此之外，如果能为开源社区的进步提供绵薄之力亦是幸运。

负责项目

我们在做中文文档社区，翻译Apache顶级的项目和国外知名的书籍，梳理和收集主流行业的学习路线图， 并且以之为乐。
我们享受、感恩并坚持开源，打破信息不对称带来的知识壁垒。在技术圈内，存在着大量的认知门槛，而这些认知门槛往往来源一语言不通而导致的信息不对称。为技术每一个爱好者，提供更加公平的环境，更加便捷的知识获取渠道，更加多元的技术内容，为不同水平，不同基础，不同目的的爱好者提供感兴趣的资料。
国内开源的现状，基本还在起步阶段，各类文档手册基本被各大平台垄断且难以维护，极少更新，很难让大家体会到开源的魅力。而且几乎全部依赖于商业利益（公司 KPI ），靠个人自愿来维护的开源项目极其稀少，个人的力量是极小的，我们一起才能走得更远。
并希望能从我们开始，一步一步提高国内开源环境的环境，最终实现 贡献 -- 讨论 -- 优化 的正向激励。
我们做过那些项目，地址：https://gitstar-ranking.com/apachecn。

赞助理由

  1. 我们做的事情得到 GitHub 认可（全球排名）。
  2. 高质量解决用户学习问题：
  1. 我们的项目有2个上万 star： 学习内容和代码这一块要求详细和精炼
  2. B站有5千多的粉丝和8万多的播放量：手把手讲解如何学习
  3. 方便更多快速学习，翻译（ GitHub 发布）前沿技术权威文档（Apache顶级项目）
  1. 在微博有5万多的粉丝，分享学习和生活上大家吐槽的一些趣事
  2. 重用和奖励贡献度大的人帮忙管理：
  1. 我们社区的贡献度和认可度非常高，也就是人气高于国内知名的个人团队。
  2. 我们组织内部也会定期参与互动和选人，保证对大家学习和生活有帮助。
  1. 录制 面试 和 项目 帮助更多人很现实的工作问题，而不是扯淡。
  2. 我们组织的构成是完全有自愿者构成，所以性质纯粹无污染。
  3. 我们组织内部管理结构式：个性化管理（不限制）+ 换届管理制度（区别于公司）。
  4. 组织内部成员很多背景一般，我发现开源不应该只是985/211，更应该是普通人。
  5. 组织内部的资料都是愿意默默无闻的贡献者，他们更多来自非常普通的公司和学校。

  3. 对热心的贡献者提供更适合的奖励（不仅仅是钱，还有工作上的技术支持）。
  4. 促进 各个外界开源组织 + 组织群内部成员 合作。
